# Fonctionnalités et Interface

### Flux de Création (Wizard)

La création d'une AREA se fait via un assistant étape par étape (`app/(tabs)/create.tsx`) immersif :

1.  **Sélection du Trigger**: Grille de services disponibles.
2.  **Configuration**: Formulaire dynamique (`ParamInputs.tsx`) basé sur les types de paramètres requis (texte, nombre, booléen).
3.  **Sélection de la Réaction**: Idem que pour le trigger.
4.  **Connexion (Review)**:
    - Une animation SVG (`react-native-svg` + `reanimated`) visualise la connexion entre les deux services (câble qui pulse).
    - L'utilisateur nomme son AREA et valide.
    - Une animation de "Confetti" confirme la création.

### Dashboard Interactif

L'écran d'accueil (`app/(tabs)/index.tsx`) liste les AREAs sous forme de cartes :

- **Swipe Gestures**: Glisser une carte vers la droite pour éditer, vers la gauche pour supprimer.
- **Toggle Switch**: Un switch animé personnalisé permet d'activer/désactiver une AREA instantanément.
- **Statistiques**: Des cartes "Glassmorphism" affichent les stats globales en haut de page.

### Gestion des Services

L'écran Services (`app/(tabs)/services.tsx`) gère deux types d'authentification :

1.  **OAuth2**: Ouvre une session `WebBrowser`. Le callback redirige vers l'app (`area://oauth-callback`) qui valide le token auprès du backend.
2.  **API Key**: Ouvre une modale pour saisir manuellement les clés (ex: Token Discord, Clé Trello).

### Gestion Offline

Un composant `OfflineNotice` surveille l'état du réseau (`expo-network`) et affiche une bannière animée si la connexion internet est perdue.
