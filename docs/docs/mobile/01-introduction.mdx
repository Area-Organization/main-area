# Introduction au Client Mobile

L'application mobile permet de créer et gérer ses automatisations AREA depuis un smartphone. Elle met l'accent sur une expérience utilisateur fluide et animée.

## Technologies Clés

- **Framework**: React Native 0.81 (via **Expo SDK 54**).
- **Langage**: TypeScript.
- **UI Library**: **React 19** (Utilisation des dernières fonctionnalités React).
- **Styling**: **NativeWind** (TailwindCSS pour React Native).
- **Animations**: **React Native Reanimated 4** pour les interactions complexes (drag, swipe, wizard).
- **Navigation**: Expo Router (Routing basé sur les fichiers).
- **API Client**: `@elysiajs/eden` (Client type-safe généré depuis le backend).
- **Authentification**: Better Auth (Client React + plugin Expo + SecureStore).

## Configuration Dynamique

Pour faciliter le développement et les tests sur appareil physique, l'application inclut un composant `UrlConfigModal`. Il permet de changer l'URL de l'API backend à la volée (stocké dans `AsyncStorage`), évitant de recompiler l'application pour changer d'environnement (localhost vs IP locale).
