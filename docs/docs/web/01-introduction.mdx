# Web Client Introduction

The web client provides a rich interface for advanced AREA management, notably through a node-based editor (Node Editor) for creating flows.

## Key Technologies

- **Framework**: SvelteKit (SSR + SPA).
- **Core Library**: **Svelte 5** (using the “Runes” syntax for reactivity, e.g. `$state`, `$derived`).
- **Language**: TypeScript.
- **Styling**: **TailwindCSS v4** + **shadcn-svelte** (Bits UI) for UI components.
- **State/Form Management**: `sveltekit-superforms` with validation via TypeBox schemas (`@area/types`).
- **Flow Editor**: **@xyflow/svelte** (Svelte Flow) for a visual “Drag and Drop” editor.
- **Authentication**: Better Auth (Svelte client).

## Structure

```text
apps/web/src/
├── lib/
│   ├── api/            # API call functions (wrappers around Eden)
│   ├── components/     # UI components (Header, ServiceCard, Forms)
│   │   ├── area-editor/# Components specific to the flow editor
│   │   └── ui/         # shadcn-svelte components
│   ├── schemas/        # Validation schemas (TypeBox)
│   └── auth-client.ts  # Better Auth client
└── routes/
    ├── +layout.svelte  # Global layout (Header, Toaster)
    ├── +page.svelte    # Dashboard (AREA list)
    ├── create/         # Creation page (Flow Editor)
    ├── login/          # Login page
    ├── profile/        # Connected services management
    └── register/       # Registration page
```
