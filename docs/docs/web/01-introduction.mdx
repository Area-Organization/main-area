# Introduction au Client Web

Le client web offre une interface riche pour la gestion avancée des AREAs, notamment grâce à un éditeur de nœuds (Node Editor) pour la création de flux.

## Technologies Clés

- **Framework**: SvelteKit (SSR + SPA).
- **Langage**: TypeScript.
- **Styling**: TailwindCSS v4 + **shadcn-svelte** (Bits UI) pour les composants d'interface.
- **Gestion d'État/Formulaires**: `sveltekit-superforms` avec validation via schémas TypeBox (`@area/types`).
- **Éditeur de Flux**: **@xyflow/svelte** (Svelte Flow) pour l'éditeur visuel de type "Drag and Drop".
- **Authentification**: Better Auth (Client Svelte).

## Structure

```text
apps/web/src/
├── lib/
│   ├── api/            # Fonctions d'appel API (wrappers autour d'Eden)
│   ├── components/     # Composants UI (Header, ServiceCard, Forms)
│   │   ├── area-editor/# Composants spécifiques à l'éditeur de flux
│   │   └── ui/         # Composants shadcn-svelte
│   ├── schemas/        # Schémas de validation (TypeBox)
│   └── auth-client.ts  # Client Better Auth
└── routes/
    ├── +layout.svelte  # Layout global (Header, Toaster)
    ├── +page.svelte    # Dashboard (Liste des AREAs)
    ├── create/         # Page de création (Éditeur de Flux)
    ├── login/          # Page de connexion
    ├── profile/        # Gestion des services connectés
    └── register/       # Page d'inscription
```
