# Fonctionnalités et Flux Utilisateur

### Authentification

-   **Inscription et Connexion**: Les utilisateurs peuvent créer un compte ou se connecter via les pages `/register` et `/login`. Ces pages utilisent `RegistrationForm.svelte` et `LoginForm.svelte`, qui gèrent la validation avec Superforms et appellent le `authClient` pour communiquer avec le backend.
-   **Gestion de Session**: `authClient.useSession()` est un store Svelte qui fournit l'état de la session utilisateur dans toute l'application.

### Dashboard (`/`)

-   La page d'accueil liste toutes les AREAs créées par l'utilisateur.
-   Elle utilise la fonction `getAreas()` (`src/lib/api/getAreas.ts`) pour récupérer les données.
-   Les utilisateurs peuvent voir le statut de chaque AREA (active/inactive) et la basculer.

### Création d'AREA (`/create`)

-   Un assistant en plusieurs étapes guide l'utilisateur :
    1.  Choix du service déclencheur (Action).
    2.  Choix de l'action spécifique et configuration de ses paramètres.
    3.  Choix du service de réaction.
    4.  Choix de la réaction spécifique et configuration de ses paramètres.
    5.  Finalisation avec un nom et une description.
-   Ce flux interactif utilise des boîtes de dialogue (`Dialog`) pour la configuration des paramètres de chaque action/réaction.

### Profil et Connexions (`/profile`)

-   Cette page permet à l'utilisateur de gérer les connexions à des services tiers.
-   Elle affiche la liste de tous les services disponibles (via `getServices`) et indique lesquels sont déjà connectés (via `getServiceConnections`).
-   Pour connecter un service, l'utilisateur clique sur une carte, ce qui le redirige vers l'URL d'autorisation OAuth2 du service. Après autorisation, il est redirigé vers la page de profil avec la connexion établie.
