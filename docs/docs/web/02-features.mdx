# Fonctionnalités

### Éditeur de Flux (Drag & Drop)

Contrairement à une approche formulaire classique, la page `/create` implémente un éditeur visuel complet :

- **Canvas Infini**: Basé sur Svelte Flow, l'utilisateur dispose d'un plan de travail.
- **Sidebar**: Liste les services disponibles (Actions à gauche, Réactions à droite).
- **Drag & Drop**: L'utilisateur glisse (`@thisux/sveltednd`) une Action et une Réaction sur le canvas.
- **Configuration des Nœuds**:
  - Les nœuds (`ActionNode.svelte`, `ReactionNode.svelte`) sont interactifs.
  - Ils affichent des champs de formulaire pour les paramètres requis.
  - Ils valident les données en temps réel.
- **Liaison**: L'utilisateur connecte visuellement le nœud Action au nœud Réaction pour valider le flux.
- **Variables**: Lorsqu'une Action est connectée, le nœud Réaction affiche les variables disponibles (`{{...}}`) pour faciliter le copier-coller.

### Dashboard

Le dashboard affiche les AREAs sous forme de grille de cartes.

- Indicateurs visuels de l'état (Actif/Inactif).
- Détails sur le dernier déclenchement.
- Possibilité de désactiver une AREA sans la supprimer.

### Profil & Services

La page `/profile` permet de gérer les connexions aux services tiers.

- Affiche l'état de connexion de chaque service (switch on/off).
- Gère la redirection OAuth2 pour connecter un nouveau service.
- Utilise les composants `ServiceCard` et `ServiceIcon` pour une présentation unifiée.
