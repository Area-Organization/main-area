# Installation and Startup

This section explains how to install, configure, and run the complete AREA project (Monorepo).

## Prerequisites

- **Runtime**: [Bun](https://bun.sh/) (v1.1+)
- **Containerization**: Docker & Docker Compose
- **Database**: PostgreSQL (if run outside Docker)

## Installation

The project is structured as a **Bun Workspaces** monorepo. Install the dependencies from the root:

```bash
# At the project root (main-area/)
bun install
```

This command will install dependencies for the backend, web, mobile, and shared packages.

## Configuration (.env)

Create a `.env` file at the root of the monorepo (main-area/.env). Below are the required variables based on the backend configuration:

```ini
# --- Database ---
# Format: postgresql://user:password@host:port/db_name
DATABASE_URL="postgresql://user:password@localhost:5432/area_db"

# --- Application URLs ---
# Used for OAuth callbacks and CORS
PUBLIC_BACKEND_API_URL="http://localhost:8080"
FRONTEND_URL="http://localhost:8081"

# --- Authentication (Better Auth) ---
# Obtain these keys via the respective developer consoles
GITHUB_CLIENT_ID="your_github_client_id"
GITHUB_CLIENT_SECRET="your_github_client_secret"

GOOGLE_CLIENT_ID="your_google_client_id"
GOOGLE_CLIENT_SECRET="your_google_client_secret"

# --- Third-Party Services (AREA) ---
# Gmail (OAuth2 Client)
GMAIL_CLIENT_ID="your_gmail_client_id"
GMAIL_CLIENT_SECRET="your_gmail_client_secret"

# Spotify (OAuth2 Client)
SPOTIFY_CLIENT_ID="your_spotify_client_id"
SPOTIFY_CLIENT_SECRET="your_spotify_client_secret"

# Trello (API Key)
# Users will provide their own token via the interface,
# but the global application API Key is required here.
TRELLO_API_KEY="your_trello_api_key"
```

## Startup

### 1. Data Base

Before launching the server, ensure the database schema is up to date via Prisma:

```bash
# Generate a Prisma client and push the schema to the DB
bun db:push
```

### 2. Development Mode

You can launch all services in parallel or individually.

**Global Launch :**

```bash
bun dev
```

**Individual Launch :**

```bash
# Backend (Port 8080)
bun run --filter '@area/backend' dev

# Client Web (Port 8081)
bun run --filter '@area/web' dev

# Client Mobile (Expo)
bun run --filter 'mobile' start
```

### 3. Testing

To run the test suite (Unit and Integration):

```bash
bun test
```
