# Introduction au Backend

Le backend est le cerveau du projet AREA. Il orchestre les authentifications, gère les connexions aux APIs tierces et exécute les tâches d'automatisation via un système de polling.

## Technologies Clés

- **Runtime**: [Bun](https://bun.sh/) - Utilisé pour ses performances et son outillage intégré (test, package manager).
- **Framework API**: [ElysiaJS](https://elysiajs.com/) - Framework haute performance avec support complet de TypeScript et validation via TypeBox.
- **Base de Données**: PostgreSQL.
- **ORM**: [Prisma](https://www.prisma.io/) - Gère le schéma de la base de données et les migrations.
- **Authentification**: [Better Auth](https://better-auth.dev/) - Gère l'authentification locale (Email/Password) et OAuth2, avec persistance via Prisma Adapter.
- **Documentation API**: Plugin `@elysiajs/openapi` pour générer une documentation Swagger conforme aux standards.
- **Tests**: `bun:test` pour les tests unitaires et d'intégration.

## Responsabilités Principales

1.  **API Gateway**: Expose les routes pour les clients Web et Mobile via `src/app.ts`.
2.  **Service Registry**: Centralise la logique des intégrations (GitHub, Gmail, Spotify, Trello, Discord).
3.  **Moteur d'Automatisation (HookManager)**: Un processus en arrière-plan qui vérifie périodiquement si les actions des utilisateurs sont déclenchées.
4.  **Gestion des Connexions**: Stocke et rafraîchit les tokens d'accès (OAuth2) ou les clés API des services tiers.
5.  **Interpolation de Données**: Permet de passer des données dynamiques de l'Action vers la Réaction (ex: `{{authorName}}` dans un message Discord).
