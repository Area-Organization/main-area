# API Endpoints

The API is documented via Swagger/OpenAPI.

### Discovery (`/about.json`)

- **`GET /about.json`**
  - Returns the list of supported services, their actions/reactions, and required parameters (type, description).
  - Used by clients to dynamically generate AREA creation forms.

### Authentication (`/api/auth/*`)

Managed by **Better Auth**. Main routes:

- `POST /api/auth/sign-up/email`
- `POST /api/auth/sign-in/email`
- `POST /api/auth/sign-in/social` (GitHub, Google)
- `POST /api/auth/sign-out`
- `GET /api/auth/session`

### Connection Management (`/api/connections`)

- **`GET /`**: Lists the services connected by the user.
- **`POST /`**: Connects a service via API Key (e.g., Trello, Discord Bot).
- **`DELETE /:id`**: Deletes a connection.
- **`GET /oauth2/:serviceName/auth-url`**: Starts the OAuth2 flow.
- **`GET /oauth2/callback`**: Standard callback for exchanging OAuth2 code.

### AREA Management (`/api/areas`)

- **`GET /`**: Lists AREAs (supports pagination and search).
- **`POST /`**: Creates a new AREA.
  - _Validation_: Ensures the user is connected to the required services and executes the action's `setup` method if needed (e.g., check if a repository exists).
- **`GET /:id`**: Retrieves details of an AREA.
- **`PATCH /:id`**: Updates an AREA.
- **`DELETE /:id`**: Deletes an AREA.
- **`POST /:id/toggle`**: Activates/Deactivates an AREA without deleting it.
- **`GET /stats/overview`**: Global statistics (number of executions, active AREAs).
