# API Endpoints

L'API est documentée via Swagger/OpenAPI.

### Discovery (`/about.json`)

- **`GET /about.json`**
  - Retourne la liste des services supportés, leurs actions/réactions, et les paramètres requis (type, description).
  - Utilisé par les clients pour générer dynamiquement les formulaires de création d'AREA.

### Authentification (`/api/auth/*`)

Géré par **Better Auth**. Routes principales :

- `POST /api/auth/sign-up/email`
- `POST /api/auth/sign-in/email`
- `POST /api/auth/sign-in/social` (GitHub, Google)
- `POST /api/auth/sign-out`
- `GET /api/auth/session`

### Gestion des Connexions (`/api/connections`)

- **`GET /`**: Liste les services connectés par l'utilisateur.
- **`POST /`**: Connecte un service via Clé API (ex: Trello, Discord Bot).
- **`DELETE /:id`**: Supprime une connexion.
- **`GET /oauth2/:serviceName/auth-url`**: Démarre le flux OAuth2.
- **`GET /oauth2/callback`**: Callback standard pour l'échange de code OAuth2.

### Gestion des AREAs (`/api/areas`)

- **`GET /`**: Liste les AREAs (pagination, recherche).
- **`POST /`**: Crée une nouvelle AREA.
  - _Validation_: Vérifie que l'utilisateur est connecté aux services requis et exécute la méthode `setup` de l'action si nécessaire (ex: vérifier l'existence d'un repo).
- **`GET /:id`**: Détails d'une AREA.
- **`PATCH /:id`**: Mise à jour.
- **`DELETE /:id`**: Suppression.
- **`POST /:id/toggle`**: Active/Désactive une AREA sans la supprimer.
- **`GET /stats/overview`**: Statistiques globales (nombre d'exécutions, AREAs actives).
