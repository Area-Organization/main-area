# Adding a New Service

The AREA project architecture is designed to be extensible. This guide explains step-by-step how to add a new service (e.g., Slack, Notion) from A to Z.

## 1. Backend

### Create the Service

Create a folder in `apps/backend/src/services/<service-name>/`.
It should contain:

- `index.ts`: The service definition.
- `actions/`: Your action files.
- `reactions/`: Your reaction files.

Example `index.ts` structure:

```typescript
import type { IService } from "../../interfaces/service";
// Import your actions/reactions here

export const myService: IService = {
  name: "myservice",
  description: "Description...",
  requiresAuth: true,
  authType: "oauth2", // or "api_key"
  // OAuth configuration or AuthFields...
  actions: [
    /* your imported actions */
  ],
  reactions: [
    /* your imported reactions */
  ]
};
```

### Register the Service

In `apps/backend/src/services/registry.ts`, import your service and add it to the registry :

```typescript
import { myService } from "./myservice";

export class ServiceRegistry {
  constructor() {
    // ... other services
    this.register(myService); // <--- Add it here
  }
}
```

Thatâ€™s it for the backend! The `/about.json` endpoint will automatically expose your new service.

## 2. Frontend Web

The web client needs an icon for your service.

### Add the Icon

Edit `apps/web/src/lib/components/ServiceIcon.svelte`. Add a condition for your service in the `if/else` block :

```svelte
<!-- apps/web/src/lib/components/ServiceIcon.svelte -->
{:else if saneName == "myservice"}
  <!-- Your SVG here -->
  <svg>...</svg>
```

## 3. Mobile Client

The mobile client uses brand colors for service tiles.

Configure the Color

Edit `apps/mobile/app/(tabs)/services.tsx`. Add your service to the `BRAND_COLORS` constant :

```typescript
// apps/mobile/app/(tabs)/services.tsx
const BRAND_COLORS: Record<string, string> = {
  discord: "#5865F2",
  // ...
  myservice: "#FF5733" // <--- Your hex color
};
```

## Summary

To add a complete service, you need to touch 3 areas:

1.  **Backend** : Implement the logic and register it (`registry.ts`).
2.  **Web** : Add the SVG icon (`ServiceIcon.svelte`).
3.  **Mobile** : Add the brand color (`services.tsx`).
