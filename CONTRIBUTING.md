# How to Contribute

Thank you for your interest in contributing to this project! We welcome contributions from the community.

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/main-area.git
   cd main-area
   ```
3. **Create a new branch** for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Making Changes

1. Make your changes in your feature branch
2. Test your changes thoroughly
3. Commit your changes with clear, descriptive commit messages:
   ```bash
   git commit -m "Add feature: description of your changes"
   ```

## Submitting Changes

1. **Push your changes** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
2. **Open a Pull Request** on GitHub with a clear title and description
3. Reference any related issues in your PR description (e.g., "Fixes #123")

## Reporting Issues

- Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md) for reporting bugs
- Use the [feature request template](.github/ISSUE_TEMPLATE/feature_request.md) for suggesting new features
- Provide as much detail as possible to help us understand and address your issue

## Code Style

- Follow the existing code style in the project
- Write clear, readable code with appropriate comments
- Keep your changes focused and avoid mixing unrelated changes

## Pull Request Guidelines

- Ensure your PR addresses a single concern
- Update documentation if you're changing functionality
- Make sure all tests pass before submitting
- Be responsive to feedback and be prepared to make changes

## Questions?

If you have questions about contributing, feel free to open an issue for discussion.

Thank you for contributing!
